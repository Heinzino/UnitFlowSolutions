```typescript
export interface PropertyMultiSelectProps {
  properties: PropertyOption[]
  selected: PropertyOption[]
  onChange: (selected: PropertyOption[]) => void
  mode?: 'single' | 'multi'  // defaults to 'multi'
  onCreateProperty?: (name: string, address: string) => Promise<PropertyOption>
  placeholder?: string
}
```

Component structure (all in one file, no sub-components):

1. **State:** `open` (boolean), `search` (string), `creating` (boolean for inline create panel), `newName` (string), `newAddress` (string), `saving` (boolean for create loading)

2. **Filtered properties:** `properties.filter(p => p.name.toLowerCase().includes(search.toLowerCase()))`

3. **Render structure:**
   - Outer `<div>` with ref for click-outside detection (same pattern as PropertySelector)
   - **Trigger button:** Shows placeholder or selected count, ChevronDown icon, `onClick={() => setOpen(!open)}`
   - **Chips section** (below trigger, always visible when selections exist): Map `selected` to chips with `<span>` + `<button onClick={() => remove(prop)}>` using X icon from lucide-react. Style: `bg-emerald/10 text-emerald border border-emerald/20 rounded-lg px-2 py-1 text-sm flex items-center gap-1`
   - **Dropdown panel** (absolutely positioned below trigger, visible when `open`):
     - Search input at top: `<input type="text" placeholder="Search properties..." value={search} onChange={...} className="w-full px-3 py-2 border-b border-gray-100 text-sm focus:outline-none" />`
     - Scrollable list (`max-h-[200px] overflow-y-auto`): Each property as a row with checkbox + name
       - Checkbox: `<input type="checkbox" checked={isSelected} onChange={() => toggle(prop)} />`
       - In single mode: selecting a new item deselects the previous one
     - Divider line
     - "Create new property" button at bottom: `<button onClick={() => setCreating(true)}>+ Create new property</button>`
     - **Inline create panel** (visible when `creating`):
       - Input: "Property name" placeholder
       - Input: "Street address" placeholder
       - Row with Cancel + Add buttons
       - On "Add" click: call `onCreateProperty?.(newName, newAddress)`, on success add to selected, reset fields, set `creating` to false
       - Show loading state on Add button while `saving` is true

4. **Click-outside handler:** Same pattern as PropertySelector — `useEffect` with `mousedown` listener, check `ref.current.contains(e.target)`

5. **Styling:** Match existing design system — white bg dropdown, `rounded-xl shadow-lg border border-gray-100`, items hover `bg-surface`, checked items have `text-emerald` accent. Use existing Tailwind tokens from the codebase.
