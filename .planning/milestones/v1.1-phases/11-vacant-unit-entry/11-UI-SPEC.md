## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, close button inset, inline badge padding |
| sm | 8px | Compact element spacing, gap between unit number + floor plan fields within a card |
| md | 16px | Default element spacing, card internal padding on mobile |
| lg | 24px | Section padding, card internal padding on desktop (`p-6` = 24px) |
| xl | 32px | Layout gaps, spacing between the unit card stack and the submit button area |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not expected in this phase) |

Exceptions:
- Touch targets for the remove (×) button on unit cards: minimum 44px hit area (use `p-2` with `min-w-[44px] min-h-[44px]` or `w-8 h-8` with negative margin).
- Page container: `max-w-2xl mx-auto py-8 px-4` — matches create-user page exactly (source: `create-user/page.tsx:32`).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F4A2A` / `#0a3a1f` body gradient | Page background (silk curtain) — set globally, not by this phase |
| Secondary (30%) | `#FFFFFF` (`--color-card`) | Unit cards, form container card, success card, property dropdown panel |
| Accent (10%) | `#22C55E` (`--color-emerald`) | Submit button (primary variant), focus rings, "Add More Units" button |
| Destructive / warning | `#EF4444` (`--color-negative`) | Validation error borders on unit cards (empty unit number or floor plan), error text below fields |
| Warning highlight | `#EAB308` (`--color-status-blocked`) | Duplicate unit number yellow border warning (not a blocker) |
| Success surface | `#f8faf6` (`--color-surface`) with `border-card-border` | Success card background tint |
| Partial failure section | `#FDE2E7` (`--color-alert-past-target`) | Failed-units sub-section within success card |

Accent (`--color-emerald`) is reserved for:
1. Submit button background (primary variant)
2. Focus ring on all inputs and dropdowns (`ring-emerald`)
3. "Add More Units" reset button
4. "Add Another Unit" secondary button (use `variant="secondary"` with emerald text — or `variant="ghost"`)

Destructive (`--color-negative`) is reserved for:
1. Unit card border when unit number is empty (validation error)
2. Unit card border when floor plan is unselected (validation error)
3. Error text below a field (`text-negative text-sm`)
4. Toast for full API failure (via sonner `toast.error()` with `richColors`)

Source: `src/app/globals.css` `@theme` block; `Input` component error state; `create-user-form.tsx` patterns.

## Layout Contract

### Page Layout
```
max-w-2xl mx-auto py-8 px-4
├── <h1> Add Vacant Units          (text-2xl font-heading font-bold text-white mb-6)
└── <AddVacantForm>
    ├── PropertyMultiSelect         (single mode, mb-6)
    │   └── Street address display  (read-only text or disabled input below selector, text-sm text-text-secondary)
    ├── Unit card stack             (flex flex-col gap-4)
    │   └── [UnitCard] × N
    │       ├── unit number Input   (flex-1)
    │       ├── floor plan <select> (flex-1, same styling as Input)
    │       └── × Button            (ghost, min 44px touch target)
    ├── "Add Another Unit" Button   (variant="secondary", w-full, mt-2)
    └── Submit Button               (variant="primary", w-full, mt-6, dynamic label)
```

### Unit Card Internal Layout
- Desktop: `flex flex-row gap-3 items-start` — unit number input and floor plan select side by side, × button on the right.
- Mobile: `flex flex-col gap-3` — unit number then floor plan stacked, × button top-right absolutely positioned or inline at top of card.

Source: CONTEXT.md decisions — "unit number input + floor plan dropdown side by side, plus a remove (×) button".

### Street Address Display
Show as read-only `<p>` below the property selector when an existing property is selected: `text-sm text-text-secondary mt-1`. When creating a new property, street address is a required Input field exposed inside the PropertyMultiSelect inline creation panel (existing behavior).

Source: CONTEXT.md — "Street address auto-fills from the selected property"; RESEARCH.md Claude's Discretion.

### Success Card Layout
```
<Card>
  ├── checkmark icon (emerald, 24px)
  ├── <h2> {N} Vacant Unit(s) Added to {Property Name}   (text-lg font-heading font-bold)
  ├── <ul> unit list: "{unit number} — {floor plan}"     (text-sm, pl-4 list-disc)
  ├── [partial failure section — bg-alert-past-target rounded p-3 mt-4]  (only if failed.length > 0)
  │   ├── "The following units could not be added:"       (text-sm font-medium text-negative)
  │   └── <ul> failed units + error message              (text-sm)
  └── "Add More Units" Button                            (variant="secondary", w-full, mt-6)
```

Source: CONTEXT.md — "checkmark, count header, property name, bulleted unit list, 'Add More Units' button"; "red section listing failed units with error messages".

### Skeleton Loading State
While `fetchProperties()` resolves (server component), the client form receives the properties array as a prop. If the page itself is loading (Suspense boundary), show:
- `<Skeleton className="h-10 w-full rounded-badge" />` in place of the PropertyMultiSelect.
- `<Skeleton className="h-[120px] w-full rounded-card mt-4" />` in place of the first unit card.

Source: CONTEXT.md Claude's Discretion — "Loading skeleton for property list fetch".

## Interaction States

### Submit Button
| State | Label | Appearance |
|-------|-------|------------|
| 0 valid units (default) | "Add Vacant Units" | disabled, opacity-50 |
| 1 valid unit | "Add 1 Vacant Unit" | enabled, primary |
| N valid units (N > 1) | "Add {N} Vacant Units" | enabled, primary |
| Submitting | "Adding units..." | disabled, spinner left of text, opacity-80 |

Source: CONTEXT.md — "dynamic label reflecting unit count"; RESEARCH.md Pattern 6.

### Validation (on submit)
- Empty unit number: `border-negative` on Input + error text "Unit number is required"
- Unselected floor plan: `border-negative` on select + error text "Floor plan is required"
- Duplicate unit number: `border-warning` (`border-status-blocked`) on affected cards + warning text "Duplicate unit number" — submission NOT blocked

Source: CONTEXT.md — "inline red border validation on submit"; "Warn on duplicate unit numbers (yellow highlight) but allow submission".

### Form Disabled State
All inputs, selects, add row button, and PropertyMultiSelect are `disabled` (pointer-events-none, opacity-50) while `submitting === true`.

### Partial Failure Recovery
After partial failure: success card appears above the form (not replacing it). Form rows reset to only the failed units pre-populated. User can correct and resubmit without re-entering property.

Source: CONTEXT.md — "Form auto-populates with only the failed unit rows for retry".

## Accessibility Notes

- Floor plan `<select>` must have an associated `<label>` (even if visually hidden using `sr-only`) for screen reader support.
- Remove (×) button aria-label: `"Remove unit {unitNumber}"` when unit number is filled; `"Remove unit row"` when empty.
- PropertyMultiSelect trigger aria-label: selected property name or placeholder text — matches Phase 10 pattern.
- Form disabled during submission: use `fieldset disabled` or per-element `disabled` prop; do not rely on visual opacity alone.
- Success card: focus moves to card heading on render (`ref` + `autoFocus` on the h2 or wrapping div with `tabIndex={-1}`).

Source: Phase 10 pattern (CONTEXT.md — "aria-label on trigger uses selected name or placeholder"); WCAG 2.1 AA standard for form controls.
