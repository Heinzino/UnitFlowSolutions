## Page Layout

The Completed Jobs page uses the `(dashboard)/layout.tsx` `AppShell` wrapper (always active for all dashboard routes). The page itself does NOT wrap in `PMDashboard` — it is a standalone page with its own header.

Page structure (matches `pm-dashboard.tsx` heading pattern exactly):

```
AppShell (from layout.tsx)
└── flex flex-col gap-4
    ├── Page header block
    │   ├── h1: "Completed Jobs"  [font-heading font-bold text-xl text-white]
    │   └── p:  "Full history of completed jobs across your properties"  [text-white/70 text-sm mt-1]
    └── Suspense (fallback: PMTurnListSkeleton)
        └── CompletedJobs (server)
            └── CompletedJobsClient (client)
                ├── PropertyMultiSelect [max-w-sm, hidden when propertyOptions.length <= 1]
                └── ActiveJobsTable
```

PropertyMultiSelect container: `max-w-sm` — prevents the filter control from stretching full width.

Source: `pm-dashboard.tsx` pattern for heading, `14-RESEARCH.md` Pattern 1 for page structure.

## Interaction Contract

### PropertyMultiSelect Behavior
- Default state: no properties selected; shows full unfiltered completed jobs list
- Selecting one property: filters table to jobs from that property only; shows chip with property name and X to remove
- Selecting multiple properties: filters table to jobs from any of the selected properties; shows one chip per property
- Deselecting all chips: returns to unfiltered full list
- Hidden entirely when `propertyOptions.length <= 1` (consistent with PM dashboard behavior)
- `onCreateProperty` prop omitted — "Create new property" button will not render

### ActiveJobsTable Sort Behavior (inherited, no changes)
- Default sort: `daysOpen` descending (longest-open first)
- Clickable column headers: Vendor (asc), Status (asc), Days Open (desc default)
- Clicking active sort column toggles direction
- Sort icons: `ChevronUp` / `ChevronDown` from lucide-react, `w-3 h-3`, `text-text-secondary`

### Navigation Entry Point
- A "View completed jobs" link is added in the `active-jobs-table.tsx` card header area, next to the "Active Jobs" heading
- Link text: "View completed jobs" — plain text link styled `text-sm text-emerald hover:underline`
- Link target: `href="/property/completed-jobs"` (no dynamic segments)
- The link appears in the header bar alongside the "Active Jobs" h2 (flex justify-between)

Source: `14-RESEARCH.md` Pitfall 4 and Pattern 4; Open Question 1 resolved as card header link.

### Loading State
- Suspense boundary wraps `CompletedJobs` server component
- Fallback: `PMTurnListSkeleton` (existing card-shaped skeleton)
- No additional loading states needed — data fetch is one-shot, filter is client-side

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "View completed jobs" — link from Active Jobs card header to `/property/completed-jobs` |
| Page heading | "Completed Jobs" |
| Page subheading | "Full history of completed jobs across your properties" |
| Filter placeholder | "Filter by property" |
| Empty state heading | "No completed jobs" |
| Empty state body | "No jobs have been marked as completed yet." |
| Error state heading | "Unable to load completed jobs" |
| Error state body | "Refresh the page or contact support if the problem persists." |
| Destructive confirmation | None — no destructive actions in this phase |
| ActiveJobsTable card heading | "Completed Jobs" — override the existing "Active Jobs" heading when rendering on this page |

Error state copy is rendered by the Next.js `error.tsx` boundary (or Suspense error path) at `src/app/(dashboard)/property/completed-jobs/error.tsx`. The boundary must render the heading and body above — do not rely on a generic "Something went wrong" fallback.

Note: The `ActiveJobsTable` currently hardcodes "Active Jobs" as the card heading. For the completed jobs page, the card heading must read "Completed Jobs." Implementation options: (a) add an optional `title` prop to `ActiveJobsTable` defaulting to "Active Jobs", or (b) accept that COMP-03 requires exact reuse and tolerate the "Active Jobs" heading. Option (a) is preferred — minimal prop addition, no behavior change.

Source: `pm-dashboard.tsx` heading pattern; `active-jobs-table.tsx` empty state copy pattern; `14-RESEARCH.md` COMP-01/COMP-02/COMP-03 requirement language.
