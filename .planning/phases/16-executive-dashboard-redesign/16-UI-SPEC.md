## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (mb-1, mt-0.5) |
| sm | 8px | Tight element spacing |
| md | 16px | Default element spacing, card padding (p-4) |

- KPI card icon container: w-10 h-10 (40px) — fixed by KPICard component

## Layout Contract

### Page Structure

```
Executive page (page.tsx) — synchronous
├── Header block
│   ├── h1: "Executive Dashboard" — text-xl font-heading font-bold text-white
│   └── p: "Welcome, {name} — {date}" — text-sm text-white/70 mt-0.5
├── Suspense fallback=<ExecutiveKPISkeleton>
│   └── ExecutiveKPIs (async server component)
│       └── div.grid grid-cols-1 sm:grid-cols-3 gap-4
│           └── [6x KPICard with footer prop]
└── Suspense fallback=<ExecutiveTop10Skeleton>
    └── ExecutiveTop10 (async server component)
        └── Card (variant="flush")
            ├── div.px-6 py-3 border-b border-border
            │   └── h2: "Top 10 Properties by Revenue Exposure" — text-xl font-heading font-bold text-text-primary
            └── Table > TableHeader + TableBody
```

Page wrapper: `div.flex flex-col gap-3` — matches existing executive page.tsx (source: CONTEXT.md + page.tsx inspection).

### KPI Grid

- Desktop: 3 columns (`grid-cols-3`)
- Mobile: 1 column (`grid-cols-1`)
- Breakpoint: `sm:` (640px)
- Gap: `gap-4` (16px)
- Row 1: Active Jobs Open | Jobs Trending Past Target | Jobs Completed (30d)
- Row 2: Backlog Delta | Avg Time to Complete | Projected Cost Exposure

Source: CONTEXT.md Decisions — KPI Box Definitions (locked).

### KPI Cards: Footer Subtitle Strings

All subtitles rendered as `<p className="text-xs text-text-secondary">` inside the KPICard footer prop.

| Card | Footer Subtitle | Null Guard |
|------|-----------------|------------|
| Active Jobs Open | `"{kpis.backlogDelta} backlog delta this week"` | None needed — backlogDelta is always a number |
| Jobs Trending Past Target | `"{riskPercent}% of active jobs at risk"` where riskPercent = activeJobsOpen > 0 ? round((jobsTrendingPastTarget / activeJobsOpen) * 100) : 0 | Safe — denominator check |
| Jobs Completed (30d) | `"{arrow} {X}% vs prior 30 days"` where arrow = ↑ or ↓ from trends.jobsCompleted.direction | Fallback: `"No prior period data"` when trends.jobsCompleted is null |
| Backlog Delta | `"More opening than closing"` or `"More closing than opening"` when backlogDelta < 0 | None needed — always a number |
| Avg Time to Complete | `"{arrow} {X}% over target · target 8 days"` from trends.avgTimeToComplete | Fallback: `"target 8 days"` when trends.avgTimeToComplete is null |
| Projected Cost Exposure | `"~$60/unit on delayed jobs"` | Static string — no guard needed |

Source: CONTEXT.md Decisions — KPI Contextual Subtitles (locked).

### Top 10 Properties Table

Container: `Card` with `variant="flush"` (removes default p-6, allows heading row to span full width).
Heading row: `div.px-6 py-3 border-b border-border` containing `h2` — matches RM property-insights.tsx pattern exactly.
Table: standard Table compound component, no sort controls (static), no clickable rows.

Columns:
| Column | Alignment | Component |
|--------|-----------|-----------|
| Property Name | text-left | Plain TableCell text |
| Revenue Exposure | text-right | CurrencyDisplay component |

Behavior:
- Sorted: Revenue Exposure descending, computed server-side
- Filter: only rows where revenueExposure > 0
- Limit: up to 10 rows (slice top 10)
- If 0 qualifying rows: render empty state (see Copywriting Contract)
- Title always reads "Top 10 Properties by Revenue Exposure" regardless of actual row count

Source: CONTEXT.md Decisions — Top 10 Properties by Revenue Exposure (locked).

## States and Interactions

| State | Where | Visual Treatment |
|-------|-------|-----------------|
| Loading — KPIs | ExecutiveKPIs Suspense boundary | ExecutiveKPISkeleton: 6 KPICard skeleton cards |
| Loading — Top 10 | ExecutiveTop10 Suspense boundary | ExecutiveTop10Skeleton: Card with 5 row skeletons |
| Empty — Top 10 | All properties at $0 revenue exposure | Render heading row + single centered row: "No properties with revenue exposure" |
| Populated — Top 10 | 1–10 properties with exposure > $0 | Standard table rows, no interactivity |
| Trend — positive/negative | KPI trend arrows | TrendIndicator handles green/red/arrow direction via trend.isGood + trend.direction |

No hover states, no click targets on table rows, no modals, no toasts — this is a read-only executive view.
