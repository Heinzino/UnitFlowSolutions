    1. **executive-kpis.ts**: Add a new exported pure function `computeKPITrends` below `computeExecutiveKPIs`:

       ```typescript
       export type TrendData = { direction: 'up' | 'down'; percentage: number } | null

       export interface KPITrends {
         jobsCompleted: TrendData
         activeJobsOpen: TrendData
         avgTimeToComplete: TrendData
       }

       export function computeKPITrends(
         currentJobs: Job[],
         prevJobs: Job[],
         currentTRs: TurnRequest[],
         prevTRs: TurnRequest[]
       ): KPITrends
       ```

       Implementation: For each of the 3 KPIs, compute the value for both windows using the same logic as computeExecutiveKPIs (reuse isCompleted, parseCurrency helpers). Percentage = `Math.abs((current - prev) / prev * 100)`. Direction = current > prev ? 'up' : 'down'. Return null when prev === 0 (no meaningful percentage).

       - **jobsCompleted**: Count Completed jobs with endDate in the respective 30d window. Current window: `now - 30d` to `now`. Previous window: `now - 60d` to `now - 30d`.
       - **activeJobsOpen**: Count non-Completed jobs. This is a point-in-time metric, so compare `currentJobs` active count vs `prevJobs` active count directly.
       - **avgTimeToComplete**: Average timeToCompleteUnit for Done TRs in each window. Current: TRs from current period. Previous: TRs from previous period. Return null if either period has 0 Done TRs.

    2. **executive-kpis.test.ts**: Add a new `describe('computeKPITrends', ...)` block with tests matching the behavior list. Use the existing `makeJob` and `makeTurnRequest` helpers. Use the same FIXED_NOW timer setup.
