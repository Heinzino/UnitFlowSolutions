  <name>Task 1: Create updateJobStatus server action with cache busting</name>
  <files>src/app/actions/job-status.ts, src/app/actions/__tests__/job-status.test.ts</files>
  <behavior>
    - updateJobStatus(jobId, turnRequestId, status) validates status is a valid JobStatus
    - On success: returns { success: true }, calls revalidateTag for job, turn request, jobs table, turn requests table, and KPIs
    - On invalid status: returns { success: false, error: "Invalid job status" }
    - On Airtable error: returns { success: false, error: "Failed to update job status" }, logs error to console
    - On job not found: returns { success: false, error: "Job not found" }
    - revalidateTag always called with { expire: 0 } (two-arg form per Next.js 16)
  </behavior>
  <action>
    1. Create `src/app/actions/job-status.ts` with `'use server'` directive at top.
       - Import `revalidateTag` from `next/cache`, `CACHE_TAGS` from cache-tags, `JOB_STATUSES`/`JobStatus` from types.
       - Import `base` and `rateLimiter` from client.
       - `updateJobStatus(jobId: number, turnRequestId: number, status: string)` returns `Promise<{ success: boolean; error?: string }>`.
       - Validate `status` is in `JOB_STATUSES` — return error if not.
       - Call `rateLimiter.acquire()`.
       - Find Airtable record: `base('Jobs').select({ filterByFormula: \`{Job ID}=${jobId}\` }).all()`.
       - If no record found, return `{ success: false, error: 'Job not found' }`.
       - Update: `await base('Jobs').update(records[0].id, { Status: status })`.
       - Cascade cache bust with `revalidateTag(tag, { expire: 0 })` for:
         - `CACHE_TAGS.job(jobId)` — the specific job
         - `CACHE_TAGS.jobs` — all jobs list
         - `CACHE_TAGS.turnRequest(turnRequestId)` — parent turn request
         - `CACHE_TAGS.turnRequests` — all turn requests list
         - `CACHE_TAGS.kpis` — KPI aggregations
       - Wrap in try/catch, log errors with `console.error('[updateJobStatus]', err)`.

    2. Create `src/app/actions/__tests__/job-status.test.ts`:
       - Mock `airtable` module — mock `base('Jobs').select().all()` and `base('Jobs').update()`.
       - Mock `next/cache` — mock `revalidateTag`.
       - Test: valid status update calls Airtable update and returns success
       - Test: invalid status returns error without calling Airtable
       - Test: job not found returns error
       - Test: revalidateTag called for all 5 cache tags with `{ expire: 0 }`
       - Test: Airtable error returns structured error (not throws)
  </action>
  <verify>
    <automated>npx vitest run src/app/actions/__tests__/job-status.test.ts --reporter=verbose</automated>
  </verify>
  <done>Server action validates input, updates Airtable, and cascade-busts 5 cache tags on success. All error paths return structured results. Tests pass with mocked Airtable.</done>
