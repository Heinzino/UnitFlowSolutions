    - fetchTurnRequests uses 'use cache' with cacheLife('airtableData') and cacheTag
    - fetchTurnRequestsForUser filters by property when role is 'pm' or 'dm', returns all for 'exec'
    - fetchJobsByIds batch-resolves using OR() filterByFormula — no N+1
  </behavior>
  <action>
    1. Create `src/lib/airtable/tables/jobs.ts`:
       - Internal `mapJob(record)` function mapping all Airtable fields to `Job` interface. Use `Number(val) || null` for numeric fields. `isCompleted` maps from checkbox/boolean.
       - `fetchJobs()` — `'use cache'`, `cacheLife('airtableData')`, `cacheTag(CACHE_TAGS.jobs)`. Calls `rateLimiter.acquire()` then `base('Jobs').select().all()`, maps with `mapJob`.
       - `fetchJobsByIds(jobIds: number[])` — `'use cache'`, tagged with `CACHE_TAGS.jobs` + individual `CACHE_TAGS.job(id)`. Returns early if empty array. Builds `OR()` filterByFormula (single ID skips OR wrapper). Calls `rateLimiter.acquire()`.
       - `fetchJobById(jobId: number)` — `'use cache'`, tagged with `CACHE_TAGS.job(jobId)`. Uses `filterByFormula: \`{Job ID}=${jobId}\``.
       - Export a `buildJobFilterFormula(jobIds: number[])` helper for testing.

    2. Create `src/lib/airtable/tables/turn-requests.ts`:
       - Internal `mapTurnRequest(record)` mapping all fields per RESEARCH.md field table. Parse `Jobs` field: if string, split by comma, map to numbers. If array, map to numbers. If null/undefined, return [].
       - Internal `resolveLinkedJobs(turnRequests: TurnRequest[])` — collects all unique jobIds across all turn requests, calls `fetchJobsByIds` once, distributes resolved jobs back to their parent turn requests.
       - `fetchTurnRequests()` — `'use cache'`, cacheLife, cacheTag. Fetches all, maps, resolves linked jobs.
       - `fetchTurnRequestById(id: number)` — `'use cache'`, tagged with record-level tag. Fetches by `{Request ID}=id`, resolves linked jobs.
       - `fetchTurnRequestsForUser(role: UserRole, assignedPropertyNames: string[])` — `'use cache'`, cacheTag. Calls `fetchTurnRequests()`, if role is 'exec' returns all, otherwise calls `filterByProperties(all, tr => tr.propertyName, assignedPropertyNames)`.

       - Test `mapJob` with mock record: verify field mapping, numeric parsing, boolean mapping
