  <behavior>
    **New PMKPIResult interface:**
    ```typescript
    export interface PMKPIResult {
      activeTurns: number
      completedThisPeriod: number
      jobsInProgress: number
      avgTurnTime: number | null
      revenueExposure: number
      revenueExposureExcludedCount: number
      turnsNearDeadline: number
    }
    ```

    **Named constants to export:**
    - `REVENUE_EXPOSURE_RATE_PER_DAY = 60`
    - `NEAR_DEADLINE_DAYS = 3`
    - `COMPLETED_PERIOD_DAYS = 30`

    **Test cases (RED phase):**

    completedThisPeriod:
    - Done TRs with readyToLeaseDate within 30 days -> counted (same logic as old completedLast30d)
    - Empty array -> 0

    jobsInProgress:
    - 3 active turns with 2 jobs each (1 Completed, 1 In Progress per turn) -> 3 (counts non-Completed, non-Ready jobs from active turns only)
    - Done turns excluded: a Done turn with an In Progress job -> 0 (job not counted because parent turn is Done)
    - Deduplication: same jobId on 2 turns -> counted once
    - Job statuses counted: 'NEEDS ATTENTION', 'Blocked', 'In Progress' are counted; 'Completed' and 'Ready' are NOT
    - Empty array -> 0

    revenueExposure:
    - Turn with offMarketDate='2024-01-01', targetDate='2024-01-11' (10 day target), daysOffMarketUntilReady=15 (5 days over) -> 300 ($60 * 5)
    - Turn not over target (daysOffMarketUntilReady=8, target span=10) -> 0 (max(0, ...))
    - Turn with targetDate=null -> excluded from sum (contributes $0)
    - Turn with offMarketDate=null -> 0 exposure (cannot compute target span)
    - Multiple turns: sum of individual exposures
    - All turns have no targetDate -> revenueExposure=0
    - Empty array -> 0

    revenueExposureExcludedCount:
    - 3 active turns, 1 has targetDate=null -> excludedCount=1
    - Done turns are NOT counted in excluded (only active turns)
    - All active turns have targetDate -> excludedCount=0
    - Empty array -> 0

    turnsNearDeadline:
    - Active turn with targetDate = today -> counted (within 3 days)
    - Active turn with targetDate = today+3 -> counted
    - Active turn with targetDate = today+4 -> NOT counted
    - Active turn with targetDate = yesterday -> NOT counted (already past)
    - Done turn with targetDate = today -> NOT counted
    - Active turn with targetDate = null -> NOT counted
    - Empty array -> 0

    activeTurns (unchanged behavior):
    - Same as existing tests

    avgTurnTime (unchanged behavior):
    - Same as existing tests

    **Existing tests that must be updated (renamed fields):**
    - `completedLast30d` test -> rename to `completedThisPeriod` (same logic)
    - `completedLast7d` test -> DELETE (field removed)
    - `projectedSpendMTD` test -> DELETE (field removed)
    - `pastTargetCount` test -> DELETE (field removed)
    - `parseCurrency` tests -> DELETE (parseCurrency removed with projectedSpendMTD)
    - Empty array edge case test -> update to check new field names
  </behavior>
  <implementation>
    After tests are RED, implement in src/lib/kpis/pm-kpis.ts:

    1. Export named constants at top of file:
       ```typescript
       export const REVENUE_EXPOSURE_RATE_PER_DAY = 60
       export const NEAR_DEADLINE_DAYS = 3
       export const COMPLETED_PERIOD_DAYS = 30
       ```

    2. Replace PMKPIResult interface with the 7-field version above.

    3. Remove `parseCurrency` helper (no longer needed without projectedSpendMTD).

    4. Update `computePMKPIs`:
       - `activeTurns`: unchanged
       - `completedThisPeriod`: same as old completedLast30d (Done TRs with readyToLeaseDate in past COMPLETED_PERIOD_DAYS)
       - `avgTurnTime`: unchanged
       - `jobsInProgress`: filter active turns (status !== 'Done'), flatMap tr.jobs, deduplicate by jobId via `new Map(jobs.map(j => [j.jobId, j])).values()`, count where `j.status !== 'Completed' && j.status !== 'Ready'`
       - `revenueExposure`: for each active turn with non-null targetDate and non-null offMarketDate, compute `targetDays = Math.ceil((new Date(tr.targetDate).getTime() - new Date(tr.offMarketDate).getTime()) / (1000*60*60*24))`, then `daysOver = Math.max(0, (tr.daysOffMarketUntilReady ?? 0) - targetDays)`, sum `daysOver * REVENUE_EXPOSURE_RATE_PER_DAY`
       - `revenueExposureExcludedCount`: count active turns where targetDate === null
       - `turnsNearDeadline`: count active turns where targetDate is not null AND targetDate is between now (start of day) and now + NEAR_DEADLINE_DAYS days (inclusive). Use: `targetMs >= todayStart && targetMs <= todayStart + NEAR_DEADLINE_DAYS * 86400000`

    5. Run `npx vitest run src/lib/kpis/pm-kpis.test.ts` to verify GREEN.
