## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, inline badge padding (`px-1`) |
| sm | 8px | Compact row gaps, between badge chips |
| md | 16px | Default element spacing, card internal padding top/bottom |
| lg | 24px | Card `p-6`, section padding, KPI grid gap |
| xl | 32px | Layout column gaps, section breaks within a card |
| 2xl | 48px | Major section separations (KPI row → Open Turns → Active Jobs) |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- KPI card internal padding: `p-4` (16px) — matches existing `KPICard` component; do not change.
- Interactive table cells (`LeaseReadyDateInput`, `DoneButton`): minimum 44px touch target height via `py-3 px-4` on the `TableCell` wrapper.
- Pill badge horizontal padding: `px-2.5` (10px) — established pattern in `pm-turn-list.tsx`; retained as-is.

Source: `src/components/ui/kpi-card.tsx`, `src/components/ui/table.tsx`, `src/app/(dashboard)/property/_components/pm-turn-list.tsx`

## Interaction Contracts

### PMDB-03: LeaseReadyDateInput

- Element: `<input type="date">` inside each Open Turn row, "Lease Ready" column
- Trigger: `onBlur` — fires once when user leaves the field; NOT `onChange`
- Guard: If `newValue === previousValue` on blur, no action taken
- Optimistic: `useOptimistic` updates displayed value immediately; reverts if server action fails
- `key` prop: `key={toInputValue(optimisticDate)}` on the input to force remount on optimistic revert
- Propagation: `onClick={(e) => e.stopPropagation()}` prevents row navigation click
- Disabled state: `disabled={isPending}` with `opacity-50` — prevents double-submit during transition
- Success feedback: `toast.success("Lease-ready date updated", { duration: 3000 })`
- Error feedback: `toast.error("Failed to update date. Please try again.")`
- Input width: `w-36` (144px) — sufficient for `YYYY-MM-DD` display in all browsers
- Styling: `text-xs border border-card-border rounded px-2 py-1 bg-card focus:outline-none focus:ring-2 focus:ring-emerald/30 disabled:opacity-50`

### PMDB-04: DoneButton

- Element: Pill button in each Open Turn row, rightmost column
- Label: "Done" (no icon — label is sufficient for single irreversible action)
- Trigger: `onClick` with `e.stopPropagation()` to prevent row navigation
- Optimistic: `useState(hidden)` — row renders `null` immediately on click; reverts (re-renders) if server action fails
- Disabled state: `disabled={isPending}` with `opacity-50`
- Success feedback: `toast.success("Turn #${requestId} marked as Done", { duration: 3000 })`
- Error feedback: `toast.error("Failed to mark turn as Done. Please try again.")`
- Styling: `text-xs bg-emerald text-white rounded-pill px-2.5 py-1 hover:bg-emerald-dark transition-colors disabled:opacity-50`
- Confirmation: No modal confirmation — the action is immediately reversible via Airtable; optimistic hide is sufficient UX
- This button is the only emerald-filled action on the Open Turns table

### PMDB-05: ActiveJobsTable Sort

- Default sort: `daysOpen` descending (oldest jobs first — most urgent)
- Clickable column headers: Vendor, Status, Days Open
- Sort indicator: `ChevronUp` (Lucide) when ascending, `ChevronDown` when descending, on the active column only
- Indicator styling: `text-text-secondary w-3 h-3 inline-block ml-1 align-middle`
- Toggle: clicking the active column reverses direction; clicking a different column sets it as active with ascending direction
- Non-interactive columns: Unit, Turn — no sort indicator rendered

### PMDB-06: Revenue Exposure Footnote

- Placement: `<p>` rendered as a sibling immediately below the Revenue Exposure `<KPICard>`, inside a `<div className="flex flex-col gap-1">`
- Condition: Only rendered when `revenueExposureExcludedCount > 0`
- Copy: `"{N} turn{plural} excluded (no target date)"` — see Copywriting Contract
- Styling: `text-xs text-text-secondary px-1`

### Open Turns — Age Column

- Column header label: "Age"
- Value: `daysOffMarketUntilReady` from `TurnRequest`, formatted as `"{N} days"`
- If null: display `"---"`
- Position: Second column in the Open Turns table, immediately after "Unit"

## Layout Structure

The PM dashboard page at `/property` renders in this top-to-bottom order after Phase 13:

```
[Property selector / header]
[KPI Row — 6 boxes in a responsive grid: 1 col mobile, 3 col sm+]
  └─ Revenue Exposure KPI: wrapped in <div flex-col gap-1> for footnote
[Open Turns section — Card]
  └─ Table: Unit | Age | Status | Lease Ready | Jobs | Done
[Active Jobs section — Card]
  └─ Table: Vendor (sortable) | Status (sortable) | Days Open (sortable) | Unit | Turn
```

Section headings within cards: `text-xl font-heading font-bold text-text-primary mb-4`

KPI grid: `grid grid-cols-1 sm:grid-cols-3 gap-4` — unchanged from existing `PMKPIs` layout.
