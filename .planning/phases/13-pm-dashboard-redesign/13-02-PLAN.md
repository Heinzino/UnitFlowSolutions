    **File 1: src/app/actions/lease-ready-date.ts**

    Create server action modeled exactly on `updateTurnRequestStatus`:
    - `'use server'` directive
    - Import `revalidateTag` from `next/cache`, `CACHE_TAGS` from `@/lib/airtable/cache-tags`, `base` and `rateLimiter` from `@/lib/airtable/client`
    - Export async function `updateLeaseReadyDate(requestId: number, date: string | null): Promise<{ success: boolean; error?: string }>`
    - Implementation:
      1. `await rateLimiter.acquire()`
      2. `const records = await base('Turn Requests').select({ filterByFormula: \`{Request ID}=${requestId}\` }).all()`
      3. If `records.length === 0` return `{ success: false, error: 'Turn request not found' }`
      4. `await base('Turn Requests').update(records[0].id, { 'Ready To Lease Date': date ?? null })`
      5. Bust 3 cache tags: `revalidateTag(CACHE_TAGS.turnRequest(requestId))`, `revalidateTag(CACHE_TAGS.turnRequests)`, `revalidateTag(CACHE_TAGS.kpis)`
      6. Return `{ success: true }`
    - Wrap in try/catch, log `[updateLeaseReadyDate]` on error, return `{ success: false, error: 'Failed to update lease-ready date' }`

    **File 2: src/app/(dashboard)/property/_components/active-jobs.tsx**

    Server component that fetches turns and extracts in-flight jobs:
    - Import `fetchTurnRequestsForUser` from `@/lib/airtable/tables/turn-requests`
    - Import `ActiveJobsTable` from `./active-jobs-table`
    - Import type `UserRole` from `@/lib/types/auth`

    Interface: `ActiveJobsProps { assignedProperties: string[]; role: string }`

    Export async function `ActiveJobs({ assignedProperties, role }: ActiveJobsProps)`:
    1. `const turnRequests = await fetchTurnRequestsForUser(role as UserRole, assignedProperties)`
    2. `const activeTurns = turnRequests.filter((tr) => tr.status !== 'Done')`
    3. `const allJobs = activeTurns.flatMap((tr) => (tr.jobs ?? []).map(j => ({ ...j, unitNumber: tr.unitNumber, turnRequestId: tr.requestId })))`
    4. Deduplicate: `const uniqueJobs = Array.from(new Map(allJobs.map((j) => [j.jobId, j])).values())`
    5. Filter per CONTEXT.md locked decision -- exclude only Completed, Invoice Sent, and Scheduled: `const inflightJobs = uniqueJobs.filter((j) => j.status !== 'Completed' && j.status !== 'Invoice Sent' && j.status !== 'Scheduled')`
       Note: Ready status jobs ARE included per user decision -- they represent in-flight workload. The JobStatus type may not include 'Invoice Sent' or 'Scheduled' but Airtable may return them as strings, so guard against them.
    6. Return `<ActiveJobsTable jobs={inflightJobs} />`
  </action>
